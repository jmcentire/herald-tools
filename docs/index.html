Registration

POST /register

Create an account and get an API key. Idempotent — same customer_id returns existing key.

Optional auth If HERALD_REGISTER_SECRET is set (self-hosted), requires Authorization: Bearer {secret}.

Request

{
  "customer_id": "my-agent",
  "ingest_auth": {             // optional
    "type": "bearer",          // or "hmac"
    "secret": "webhook-secret" // bearer: shared secret
    // hmac: "key" + "header"
  }
}

Response 201 Created (new) / 200 OK (existing)

{
  "object": "account",
  "customer_id": "my-agent",
  "api_key": "hrl_sk_...",
  "created": 1775183297
}

HTTP status discriminates new (201) from existing (200); created is the Unix-second timestamp of the original registration.

customer_id constraints: 3–32 chars, [a-z0-9-] only, must start with a letter or digit, must not collide with reserved top-level routes (account, ack, admin, api, billing, dlq, docs, endpoints, health, heartbeat, messages, nack, queue, register, stream, stripe, ...).

Rate limit: per-IP 5/minute and 50/day. Source IP read from X-Forwarded-For or X-Real-IP.

Ingest Auth Options

TypeFieldsHow it works
bearersecretProvider sends Authorization: Bearer {secret}
hmackey, headerProvider signs body with HMAC-SHA256, puts signature in named header. Herald validates. Handles sha256= prefix, hex and base64.

Configuration Options

FieldDefaultDescription
encryption"service""service" (AES-256-GCM) or "none" (plaintext). Secure by default.
retention_daysTier defaultOverride retention (1–90). Capped by tier max (Free: 7d).
{
  "customer_id": "my-agent",
  "config": {
    "encryption": "none",
    "retention_days": 3
  }
}

Ingest (Webhook Providers)

POST /{customer_id}/{endpoint_name}

Receive a webhook. No authentication required unless ingest_auth is configured for this customer.

Body: raw webhook payload (any content type). Headers are preserved.

Response 200 OK

{
  "object": "ingest_result",
  "message_id": "msg_a1b2c3...",
  "fingerprint": "fp_d4e5f6...",
  "received_at": 1775183297,
  "received_at_ns": "1775183297820527578"
}

received_at is Unix integer seconds. received_at_ns is a decimal string of nanoseconds (does not fit in 32-bit; many JSON libs cannot decode 64-bit integers natively).

Deduplication

If the same body was already received for this endpoint:

{
  "object": "ingest_result",
  "fingerprint": "fp_d4e5f6...",
  "deduplicated": true
}

Polling (Agents)

GET /endpoints/{endpoint_name}/messages Auth required

Fetch queued messages. Messages become invisible for visibility_timeout seconds.

Query Parameters

ParamDefaultDescription
limit10Max messages to return (1–100)
visibility_timeout300Seconds before redelivery (30–43200)

Response 200 OK

{
  "object": "list",
  "data": [
    {
      "object": "message",
      "message_id": "msg_a1b2c3...",
      "fingerprint": "fp_d4e5f6...",
      "body": "base64-encoded-payload",
      "headers": {"Content-Type": "application/json"},
      "received_at": 1775183297,
      "deliver_count": 1,
      "encryption": "service"
    }
  ],
  "has_more": false,
  "queue_depth": 0
}

Empty queue returns the same envelope with data: [] and has_more: false. There is no 204 branch.

Acknowledge / NACK / Heartbeat

POST /endpoints/{endpoint_name}/messages/{message_id}/ack Auth required

Mark a message as processed. Removes it from the queue.

{"object": "ack_result", "message_id": "msg_a1b2c3...", "acknowledged": true}
POST /endpoints/{endpoint_name}/messages/ack Auth required

Batch acknowledge. Body: {"message_ids": ["msg_...", "msg_..."]}

{"object": "batch_ack_result", "acknowledged": ["msg_..."], "failed": []}
POST /endpoints/{endpoint_name}/messages/{message_id}/nack Auth required

Reject a message. Body selects the disposition (defaults to requeue if body omitted).

// Body
{"disposition": "requeue"}   // or "dlq"

// Response
{"object": "nack_result", "message_id": "msg_...", "disposition": "requeue"}

disposition is an open enum — future values may include delay_requeue or discard. Unknown values are rejected with 400.

POST /endpoints/{endpoint_name}/messages/{message_id}/heartbeat Auth required

Extend visibility timeout. ?extend=600 (seconds, 30–43200).

{"object": "heartbeat_result", "message_id": "msg_...", "visibility_timeout_extended": true, "extended_by": 300}

WebSocket Streaming

GET /endpoints/{endpoint_name}/stream

Upgrade to WebSocket. First message must be auth: {"type": "auth", "api_key": "hrl_sk_..."}

Server sends {"type": "message", "message_id": "msg_...", ...} as messages arrive. Client sends {"type": "ack", "message_id": "msg_..."} or {"type": "nack", "message_id": "msg_...", "disposition": "requeue"}.

Error Responses

All errors share the shape:

{
  "error": {
    "type": "rate_limit_exceeded",
    "code": "queue_depth_exceeded",
    "message": "queue depth limit reached; ACK existing messages to make room"
  }
}
Statustype / codeMeaning
400invalid_request / bad_requestInvalid customer_id, malformed body, unknown enum value.
401authentication_error / unauthorizedMissing or invalid API key, or invalid ingest auth.
404invalid_request / not_foundMessage not in flight (already acked, expired, or never existed).
413invalid_request / payload_too_largeBody exceeds tier limit (Free: 64KB, Standard: 1MB, Pro: 10MB).
429rate_limit_exceeded / rate_limit_exceededDaily/burst quota exceeded. Retry-After header included.
429resource_exhausted / queue_depth_exceededEndpoint queue is full; ACK existing messages to make room. Retry-After header included.
500internal_error / internal_errorUnexpected server failure.

Authentication

Agent Auth (polling, ack, nack, heartbeat)

Send your API key as a Bearer token: Authorization: Bearer hrl_sk_...

Get your key from POST /register.

Ingest Auth (optional, per-customer)

Configure via the ingest_auth field when registering. Two modes:

ModeProvider sendsHerald validates
BearerAuthorization: Bearer {secret}Constant-time comparison
HMACSignature in named header (e.g., X-Hub-Signature-256: sha256=...)HMAC-SHA256 of body with stored key

If no ingest_auth configured: ingest is open (default).

Register Auth (self-hosted only)

Set HERALD_REGISTER_SECRET env var. If set, POST /register requires Authorization: Bearer {secret}.
